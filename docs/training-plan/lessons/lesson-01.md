# 第 1 讲：美赛流程、题型与团队分工

## 课程定位

这一讲是整个培训的启动课。目标是让学员先理解 MCM/ICM 的完整交付链，而不是一上来只学软件。

## 课堂目标

- 理解 MCM/ICM 的时间线、提交物和评分关注点
- 区分数据手、建模手、编程手、论文手的职责
- 能把一道题拆成子问题、数据需求和模型候选

## 核心概念

- 题目拆解
- 队伍分工
- 交付物管理
- 建模流程
- 论文证据链

## 课堂演示

- 展示一份典型赛题，拆出背景、目标、约束、数据和输出
- 展示比赛期间 4 天时间安排
- 展示代码、图表、论文之间的引用关系

## 课堂练习

给一段简化赛题材料，要求学员完成：

- 子问题拆解
- 数据需求列表
- 可能模型列表
- 队内角色分工

## 课后作业

选择一个历年赛题，提交 1 页题目分析：

- 问题背景
- 子问题拆解
- 数据需求
- 初步模型选择
- 队伍分工方案

## 验收标准

| 项目 | 标准 |
| --- | --- |
| 问题拆解 | 至少拆出 3 个明确子问题 |
| 数据需求 | 能说明每个子问题需要什么数据 |
| 模型选择 | 模型与问题类型匹配 |
| 分工方案 | 角色职责清楚、可执行 |

## 后续建设建议

补充一份历年优秀论文拆解案例，作为本讲示范材料。
