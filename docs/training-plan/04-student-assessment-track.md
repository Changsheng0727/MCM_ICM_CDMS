# 04 学员作业与评估线

## 这条线解决什么

学员作业与评估线关注的是：学生是否真的学会，而不是是否听完课程。

每讲都要有一个明确交付件。交付件最好能直接模拟美赛中的某个角色任务：

- 数据手
- 建模手
- 编程手
- 论文手

## 作业设计原则

1. 每次作业只考核 1-2 个核心能力
2. 作业必须有输入数据或明确问题
3. 作业必须要求图表或表格输出
4. 作业必须包含简短文字解释
5. 评分标准提前给出

## 第 2 讲作业

### 主题

数据导入、清洗、统计与可视化。

### 学员交付

- 清洗后的数据文件
- 描述统计表
- 至少 3 张图
- 200 字以内的数据质量说明

### 评分标准

| 项目 | 分值 |
| --- | --- |
| 数据导入正确 | 20 |
| 缺失值处理合理 | 20 |
| 描述统计完整 | 20 |
| 图表清晰 | 20 |
| 解释准确 | 20 |

## 第 3 讲作业

### 主题

回归、聚类和 PCA 建模。

### 学员交付

- 回归模型结果和解释
- 聚类结果和解释
- PCA 可视化图
- 一段模型局限性说明

### 评分标准

| 项目 | 分值 |
| --- | --- |
| 模型选择合理 | 20 |
| 代码能运行 | 20 |
| 图表和结果清晰 | 20 |
| 解释有实际含义 | 25 |
| 能说明局限性 | 15 |

## 第 4 讲作业

### 主题

资源分配或生产计划优化。

### 学员交付

- 决策变量定义
- 目标函数
- 约束条件
- MATLAB 求解代码
- 最优解解释

### 评分标准

| 项目 | 分值 |
| --- | --- |
| 决策变量定义清楚 | 20 |
| 目标函数正确 | 20 |
| 约束条件完整 | 25 |
| 求解代码能运行 | 20 |
| 结果解释合理 | 15 |

## 综合模拟任务

建议在 4-6 讲后安排一次 3-5 天的小型模拟赛：

1. 给一份题目和数据
2. 学员分组完成数据处理、模型建立、求解和短报告
3. 每组提交一份 4-6 页报告
4. 讲师按建模完整性、可复现性、图表质量、解释质量评分

## 学员成长档案

建议给每位学员记录：

- 每讲作业完成情况
- 代码可运行性
- 图表质量
- 文字解释能力
- 最适合承担的队内角色
