# 05 路线选择与执行节奏

完整 18 讲课程规划见：[06-18-lesson-syllabus.md](06-18-lesson-syllabus.md)。

## 路线 1：课件补全优先

适合当前目标是尽快把培训课程补齐、上线、能讲。

配比：

- 授课交付线 `40%`
- Python/Jupyter 代码与复现线 `30%`
- 建模方法线 `20%`
- 学员作业与评估线 `10%`

近期重点：

- 把第 2、3、4 讲整理成标准课程结构
- 为每讲补作业和验收标准
- 更新 README，让学员知道如何使用仓库

## 路线 2：代码复现优先

适合当前目标是让仓库从课件展示变成可运行、可复现的训练库。

配比：

- Python/Jupyter 代码与复现线 `45%`
- 建模方法线 `25%`
- 授课交付线 `20%`
- 学员作业与评估线 `10%`

近期重点：

- 将早期 `.mlx` 思路迁移为可复用 `.py` 脚本和 Notebook 单元
- 建立 `data/`、`src/`、`outputs/` 结构
- 确保关键图表和结果表能保存

## 路线 3：模拟赛训练优先

适合当前已有基础课件，准备训练学生完成完整比赛流程。

配比：

- 建模方法线 `35%`
- 学员作业与评估线 `30%`
- Python/Jupyter 代码与复现线 `20%`
- 授课交付线 `15%`

近期重点：

- 设计一个小型综合题
- 学生分组完成数据处理、模型、图表和报告
- 讲师按完整建模流程评分

## 8 周建设节奏

### 周 1-2：课程结构整理

目标：

- 梳理已有 Notebook 和早期 `.mlx` 参考课件
- 将第 1-6 讲补齐目标、练习、作业、验收标准

交付：

- 第 1-6 讲课程设计表
- 每讲 1 个课后作业

### 周 3-4：代码复现整理

目标：

- 建立可复现目录结构
- 从 Notebook 中抽取关键 Python 脚本
- 保存结果图表

交付：

- `data/`、`src/`、`outputs/` 初版结构
- 至少 3 个可运行 `.py` 脚本或 Notebook 示例

### 周 5-6：模型模块补强

目标：

- 补齐第 7-14 讲中的评价、预测、分类、图模型、仿真和检验模块
- 为模型配作业和解释模板

交付：

- 至少 4 个新增模型专题设计
- 每个专题 1 个对应作业
- 每个专题 1 份结果解释范例

### 周 7-8：小型模拟赛

目标：

- 完成第 15-18 讲的论文表达与综合模拟赛训练

交付：

- 题目与数据
- 学员提交报告
- 讲师评分表
- 复盘总结

## 每周固定动作

- 周一：确定本周要补的课程或代码模块
- 周二：完善课件和示例
- 周三：跑通代码和输出图表
- 周四：设计作业和评分标准
- 周五：更新文档和 README
- 周末：复盘本周建设质量

## 最小周交付

每周至少完成：

- 1 个课件或模型模块改进
- 1 个可运行代码或输出文件
- 1 份作业、评分表或复盘记录
